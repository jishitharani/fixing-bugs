## 🐛 Fixing Bugs

Fixing bugs is an important part of improving this project and helping the community. A bug is any error, flaw, or unexpected behavior in the code that affects how the program works.

### 🎯 Objective

Identify and resolve issues to make the project more stable, efficient, and user-friendly.

### 🛠️ What You Can Do

* Find and fix coding errors
* Correct logical mistakes
* Improve performance issues
* Fix UI/console output problems
* Update outdated or incorrect code

### 📌 How to Contribute

1. Check the Issues section for reported bugs
2. Choose a bug labeled `easy`, `medium`, or `critical`
3. Fork the repository
4. Fix the issue in the code
5. Submit a Pull Request with a clear explanation

### 💡 Contribution Guidelines

* Clearly describe what was wrong
* Explain how you fixed it
* Keep your code clean and readable
* Test your fix before submitting

### 🏆 Rewards

* Earn XP based on difficulty
* Get featured in the leaderboard
* Receive recognition in the project

### 🤝 Why It Matters

Fixing bugs helps everyone. Your contribution improves the project for all users and supports other developers. In return, you gain experience, recognition, and community support.
